# 掼蛋 · Guandan（H5 离线版）

一个完整的掼蛋游戏。**离线优先**（无网络可玩，适合飞机/地铁），基于纯 H5（HTML/CSS/原生 JS，无构建、无框架、无第三方依赖），引擎与界面解耦，便于后续做联机版与多端迁移。

## 运行方式
- **在线（PWA · 主屏 App）**：打开 **https://anyweil.github.io/guandan/** ，手机浏览器「添加到主屏幕」即可像原生 App 一样独立全屏运行、断网可玩（首次打开后自动缓存全部资源）。
  - iOS：Safari → 分享 → 添加到主屏幕。Android：Chrome → 菜单 → 安装应用 / 添加到主屏幕。
- **离线（单文件）**：双击 `index.html` 或自包含的 `guandan.html`（`file://` 下运行，无需服务器、无需联网）。
- **引擎测试**：`node test/engine.test.js`

## 部署（GitHub Pages）
仓库根目录直发 `main` 分支（已含 `.nojekyll`、`manifest.webmanifest`、`sw.js`、`icons/`）：
```
git push origin main          # 推送即自动发布到 https://anyweil.github.io/guandan/
```
图标如需重绘：`python3 icons/gen_icons.py`（依赖本机 PIL 与 PingFang 字体）。

## 规则口径
采用**全国竞技掼蛋标准规则**：
- 4 人 2v2，对家为队友；双副牌共 **108 张**（含 2 小王 + 2 大王）。
- 每人 27 张；**级牌**从 2 起打；**红桃级牌为「逢人配」**（万能牌，可代除王外任意牌）。
- 牌型：单 / 对 / 三 / 三带二 / 顺子(5张) / 三连对(木板,6张) / 二连三(钢板,6张) / 炸弹(4+张) / 同花顺 / 天王炸(四王)。
- 炸弹大小：四张炸 < 五张炸 < **同花顺** < 六张炸 < … < 八张炸 < 天王炸；同级比点数。级牌点值抬至 A 之上、王之下。
- 升级 / 进贡还贡 / 抗贡 / 接风 等流程在阶段②实现。
> 如需地方变体（如不同炸弹序、不同升级规则），改 `engine.js` 常量与比较函数即可。

## 怎么玩
1. 双击 `index.html` 打开（离线即可）。默认你坐南、对家(北)=高级、左右对手=中级，从打 2 开始。
2. 轮到你：点牌选中（金色抬起）→「出牌」；跟不起点「不要」；不会出点「提示」（可连点切换候选）。
3. **🎴 理牌**：一键把手牌整理成「最少手数」的最优牌型组合（炸弹/同花顺/钢板/连对/顺子/三带二/三/对/散张分组展示），并提示「整理为 N 手可走完」。
4. **🤖 托管**：一键让 AI 替你打，随时可「取消托管」收回操作。
5. 想改难度/起始级牌：点右上「⚙ 设置」，分别给对家/上家/下家设档（入门→宗师），「开始新整场」。
6. 一路打到 A 并在 A 上拿头游即「过 A」获胜。

## AI 五档设计（阶段③ · 记忆驱动重构）
**统一核心**（所有档位不变的大原则）：让**自己的出牌手数最少**以抢头游；牌力差则转为帮队友头游；同时尽量让对手多花手数、拿不到牌权。每手出牌都力求**有回手 / 让队友接管 / 让对手接不动（除非放炸）**，从而保住牌权。AI 每回合按当前记忆**动态重估**最优出牌组合（玩家手工理牌，AI 自动调整）。

**唯一的档位差异 = 记忆能力**（记得越多→越能识别绝张/控场→越敢领小牌控场、越省炸）：
| 档位 | 记忆能力 |
|---|---|
| 入门 | 无记忆（不记牌）。 |
| 中级 | 记住大小王 + 主牌（级牌）已出的张数与花色。 |
| 高级 | + 记住所有 10 以上的牌（已出张数与花色）。 |
| 大师 | 记住所有已出的牌（张数与花色）。 |
| 宗师 | + 记住每张是谁出的（据此评估对手强弱，放强打弱、省炸防真威胁）。 |

**出牌口径**：单/三带二须有回手或队友能接才出；顺子留后（对手越打越接不动）；木板/钢板早出（对手难跟）。
**跟牌纪律（记忆的主战场）**：最小可压若压不死对手（非绝张、会被反压）又不紧迫，宁可 pass 留牌；能确认是绝张才跟下去赢墩夺权。记忆越准 → 越分得清「值得跟的绝张」与「白送的大牌」。
**出炸口径**：任一对手可能 1 手走完→必炸；对手控场且剩 ≤5 张→炸夺权；威胁感由记忆估算（记忆差→以为外面全是大牌→虚高→滥炸浪费）；强弱取舍可放掉不紧迫的强对手、省炸专防紧迫的弱对手。

**强度如何验证**：掼蛋 = 运气 + 水平，普通对局里发牌运气会淹没水平差异。故用**复式对局**评测——同一副牌让强弱两方各坐一遍（运气完全抵消），只比「拿同一手牌时谁更常拿头游」。结果：**有记忆各档 vs 无记忆的入门 ≈ 55%–56%（明显更强）**；相邻记忆档之间约 50%–51.5%。这反映真实掼蛋的规律：**「会不会记牌」差距明显，而「记多记少」的边际收益在单局里很小**。

## 设计原则
- **引擎纯函数、可测试、与 UI/AI 解耦**：`engine.js` 不依赖 DOM，Node 与浏览器通用。
- **离线零依赖**：普通 `<script>` 加载，`file://` 可跑。
- **可复现**：发牌支持随机种子，便于调试与复盘。
